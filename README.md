1. **崩溃证据捕获 (1.18)**：
   - 强化日志处理器：增加 `QMutex` 确保多线程安全，每条日志写入后执行 `flush()`，防止闪退导致线索丢失。
   - 自动化资源监控：在 `MainWindow` 集成 5 秒周期的定时器，实时记录 GDI 对象、USER 句柄及私有内存占用，以便定位句柄泄漏或 OOM。
   - 修正 WER 脚本：确保针对 `ArcMeta.exe` 生成完整 Dump。

2. **导入逻辑归一化 (1.19)**：
   - 构建 `ImportHelper` 模块：统一了“扫描”与“拖拽”的处理流，实现“扫描即镜像”的行为对齐。
   - 增强反馈与性能：进度条支持 `[当前/总数] % - 文件名` 精细格式，引入 500 项批次物理存盘策略。
   - 引入中断保护：关闭进度框时弹出二次确认，支持“保留进度”或“一键撤销（ImportCommand）”。

3. **单实例运行限制 (1.20)**：
   - 利用 Windows Mutex ("ArcMeta_SingleInstance_Mutex") 与 `QLockFile` 双重机制，防止多开实例引发的数据库损坏。

4. **安全退出与占用解决 (1.21)**：
   - 重构退出链路：托盘退出时弹出模态提示。
   - 步进式持久化：将 `flushAll` 改为 `sqlite3_backup_step`（每 50 页一跳），并在跳跃间隙处理事件循环以维持 UI 响应。
   - 物理句柄清理：显式调用 `sqlite3_close_v2` 释放所有 DB 连接，解决文件占用无法删除的故障。

5. **底层加固**：
   - 为 `SqlTransaction` 增加了 5 次重试（50ms 间隔）的 `SQLITE_BUSY` 处理逻辑，增强高并发扫描时的事务稳定性。
