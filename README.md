此提交解决了以下核心 UI 和数据一致性问题：
1. **消除 UI 闪烁**：通过移除 MainWindow 中的冗余异步调用并在 ContentPanel 中添加基于状态的守卫，我们停止了导致分类切换期间视觉闪烁的“双重清空”效应。
2. **统一渲染**：此前，搜索结果和特定路径加载绕过了一些元数据注入（如图像尺寸），导致卡片渲染不一致（缩略图未填充）。我们现在使用集中的 createItemRecord 工厂方法。
3. **基于 FID 的一致性**：不同位置的相同文件现在通过使用 NTFS 文件 ID (FID) 作为唯一标识符（而非仅路径）来共享元数据和图标缓存资源。
4. **改进搜索 UX**：清空搜索栏现在可以正确地将视图回滚到之前活动的目录。
