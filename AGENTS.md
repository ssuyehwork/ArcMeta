# AGENTS.md — Jules 行为准则
本文件是 Jules 的强制行为规范。所有规则必须严格遵守，不得跳过、简化或以"效率"为由绕过任何步骤。

# 1. 任何时候都必须留意或记住这句话“你先描述一下你对这段话的理解,是否达成共识,避免误解而导致浪费时间”，主打一个以灵活探讨 → 达成共识 → 授权修改
# 2. 任何一次用户下达的新任务时，你在接收到新任务之后，必须自述一遍，避免误解而浪费时间。若用户没有明确回复"理解正确"或等价确认，Jules 不得视沉默为同意而自行推进。

# 3. 修改代码准则
## 3.1 禁止擅自扩大修改范围，禁止越界修改代码
## 3.2 修改代码时，如果发现代码的逻辑存在风险、不合理、不规范，那么就应该将相关问题申报备案到“Declaration_Log.md”文档里，后期再由用户来下指令修改/调整，而不该脑补修改 / 顺手修改
## 3.3 修改代码时，务必结合上下文来修改代码，每次务必做到开箱即用。
