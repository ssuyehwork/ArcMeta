# 用户需求记录 (User_Requirement_Record.md)

## 1. 核心需求：托管库物理与逻辑深度联动

### 1.1 场景一：同步创建
- **触发点**：用户在盘符栏右键菜单选择“创建托管文件夹”。
- **物理行为**：在对应盘符根目录创建名为 `ArcMeta.Library_[盘符]`（如 `Z:/ArcMeta.Library_Z`）的文件夹。
- **期望逻辑行为**：**同步**在侧边栏“我的分类”中创建一个同名的逻辑分类，确保两者物理与逻辑上一一对应。

### 1.2 场景二：缺失补全 (物理驱动逻辑)
- **触发点**：系统检测（可能是启动时或运行中感知）。
- **现象**：磁盘根目录下已存在符合命名的物理文件夹 `ArcMeta.Library_[盘符]`，但侧边栏“我的分类”中不存在对应的逻辑分类。
- **期望行为**：系统应自动为该物理路径补全相应的逻辑分类。

- 确认当前代码实现状态，并设计符合“物理驱动逻辑”目标的架构演进路径。
