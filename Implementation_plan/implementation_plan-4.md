# 入库状态标识误报与视觉欺诈专项修复 —— implementation_plan-4.md

## 1. 任务背景
修复托管库项目在未登记、未入库的情况下，卡片右上角错误显示绿色对勾标识的“视觉欺诈”问题。

## 2. 问题定位
- **位置**：`src/meta/MetadataManager.h:46` 及 `src/ui/ThumbnailDelegate.cpp:153`。
- **根因分析**：
    1. `RuntimeMeta` 构造函数违规将 `ingestionStatus` 默认初始化为 `1` (Ingested)。
    2. `ThumbnailDelegate` 在获取不到模型数据时，默认降级取值为 `1`。
- **后果**：只要项目物理位于托管库下，即便数据库中无记录，UI 也会直接假定其已入库（对应用户质疑："数据库里根本就没有被登记，为何却显示这个标识呢？"）。

## 3. 强制对照表

| 编号 | 用户原话 / 我的理解 | 方案对应点 | 是否一致 |
|------|---------------------|------------|----------|
| 1    | 卡片右上角标识意味着项目已经入库了对不？ | 确认标识含义，并揭露误报根源 | ✅ |
| 2    | 数据库根本没有被登记，为何却显示标识 | 修正默认值逻辑，严控显示前置条件 | ✅ |

## 4. 详细解决方案
- **4.1 构造函数拨乱反正**：修改 `RuntimeMeta` 默认构造函数，将 `ingestionStatus` 初始值由 `1` 修正为 `-1` (Invalid/Unknown)。
- **4.2 UI 渲染逻辑加固**：
    - 修改 `ThumbnailDelegate::paint`：取消默认取值 `1` 的逻辑，改为默认取值 `-1`。
    - **显示判定约束**：只有当 `ingestionStatus == 1`（代表数据库明确记录为已入库）且物理路径在库内时，才允许绘制绿色对勾。
- **4.3 状态位同步**：确保 `registerItem` 动作发生时，初值显式设为 `0` (Registered)，严禁跳级。

## 5. 修改边界声明【红线】
**涉及范围**：`MetadataManager.h` 构造函数与 `ThumbnailDelegate.cpp` 绘图判定。
**禁改范围**：禁止修改 `ThumbnailDelegate` 的图标资源或布局参数。

## 6. 实现准则与预警
- **预警**：修改默认值后，未入库项目的对勾将立即消失并转为蓝色进度弧（若已登记）或无标识（未登记），这符合系统真实状态。
