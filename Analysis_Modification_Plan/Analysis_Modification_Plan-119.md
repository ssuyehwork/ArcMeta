4. 详细解决方案
### 4.1 核心状态自愈 (Ingestion Logic)
- **废除内存硬编码**：删除审计出的所有手动设置 ingestionStatus 的语句（具体数量及位置见待确认事项）。
- **事务驱动状态**：状态更新必须由数据库事务成功提交信号驱动。
- **物理清理僵尸 ID**：从 Repo 及菜单中移除所有相关代码路径（具体 ID 见待确认事项）。

### 4.2 业务逻辑物理剥离 (Logic Offloading)
- **职责迁移**（对应用户原话：“怎么办呢？”）：将磁盘库创建逻辑与硬件消息捕获迁移至 `CoreController` 或专职 Service。
- **搜索历史管理**：封装独立单例负责逻辑维护。

### 4.3 通讯与事件重构 (Decoupling)
- **搜索状态闭环**：搜索输入必须先同步至筛选面板。
- **事件过滤器胶束化**：将 Resize、Hover 等逻辑拆分到独立派生类。
- **清理冗余调用**：物理删除构造函数中的冗余置顶代码（具体行号见待确认事项）。

### 4.4 物理环境治理 (Environment)
- **强制对齐小写**：物理重命名违规文件夹。
- **方案入库**：强制移动根目录散落文件。
