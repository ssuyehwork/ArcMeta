# 技术架构与逻辑优化方案：内容容器排序权重优化 (Analysis_Modification_Plan-37.md)

## 1. 排序逻辑优化：子分类/子文件夹置顶

### 1.1 现状分析
当前 `FilterProxyModel::lessThan` 仅处理了 `PinnedRole`（置顶）。当用户按名称、日期或大小排序时，子分类（Category）或物理文件夹（Folder）会混入普通文件之中，破坏了文件管理的层级感和操作效率。

### 1.2 解决方案：注入 `isCategory` 与 `isDir` 权重
必须修改 `lessThan` 函数，建立三级排序权重体系，确保视觉展示的层级分明：

1.  **权重 I (置顶状态)**：
    -   检查 `PinnedRole` 或 `IsLockedRole`。
    -   被置顶的项具有绝对优先权。

2.  **权重 II (数据类型 - 置顶)**：
    -   在非置顶项中，或者同为置顶项时，判断是否为“分类”（`isCategory`）或“文件夹”（`isDir`）。
    -   **规则**：子分类和文件夹始终优于普通文件。
    -   **实现**：从 `source_left.data(TypeRole)` 获取类型，或直接访问 `FerrexVirtualDbModel` 中的 `isCategory/isDir` 标记。

3.  **权重 III (用户排序列数据比较)**：
    -   在权重 I 和 II 均相等的情况下，才执行标准的数据比较（名称、日期、大小等）。
    -   必须结合 `sortOrder()` 进行逻辑判定，确保在降序模式下，置顶项和分类项依然保持在列表顶部。

### 1.3 预期效果
无论用户选择升序还是降序，列表结构将始终保持：
-   [置顶项]
-   [子分类 / 文件夹]
-   [普通文件]

这种“文件夹优先”的排序规则符合 Windows Explorer 及专业管理工具的通用标准，极大提升了用户在大规模数据集下的导航体验。
