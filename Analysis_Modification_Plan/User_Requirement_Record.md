# 用户需求记录 User_Requirement_Record.md

## 2026-07-xx 架构演进：磁盘优先数据流向规划

### 1. 需求背景
当前程序退出时存在明显延迟，原因是系统采用“内存数据库 + 退出时同步磁盘”的架构，导致退出前必须执行大量的磁盘 IO。

### 2. 用户原话
“当前版本，我期望先将数据插入到数据库然后更新内存，凡是与此不符的都必须做调整，这样在退出的时候就可以秒推出了”

### 3. 核心目标
- 翻转数据流向：由“内存优先”转为“磁盘库优先”。
- 物理数据库实时保持最新状态。
- 实现程序“秒退出”，彻底移除退出时的同步循环逻辑。

### 4. 涉及模块
- `DatabaseManager`: 废弃内存数据库中转，直接连接物理文件。
- `MetadataManager`: 重构持久化逻辑，确保写入磁盘成功后再同步内存镜像。
- `CategoryRepo`: 调整分类关联的事务处理，确保物理库一致性。
- `TrayController`: 裁剪退出逻辑。
