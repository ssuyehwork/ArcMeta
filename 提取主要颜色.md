> **Implement a C++ command-line tool that extracts dominant colors from PSD, AI, and EPS design files.**
>
> **Workflow:**
> 1. Accept a design file (`.psd`, `.ai`, or `.eps`) and an optional color count (default: 6) as CLI arguments
> 2. Convert the input file to PNG first:
>    - `.psd` → use ImageMagick (`magick input.psd[0] -flatten output.png`)
>    - `.ai` / `.eps` → use Ghostscript (`gs -dNOPAUSE -dBATCH -dSAFER -sDEVICE=png16m -r150 ...`)
> 3. Load the PNG with OpenCV, convert to Lab color space, and run K-Means clustering to find the N dominant colors
> 4. Filter out near-white and near-black background pixels (HSV saturation < 20) before clustering
> 5. Sort results by descending pixel percentage and print them in a formatted table showing rank, HEX value (#RRGGBB), and coverage percentage
>
> **Requirements:**
> - Language: C++17
> - Dependencies: OpenCV (for image processing + K-Means), ImageMagick CLI, Ghostscript CLI
> - Build system: CMake (`CMakeLists.txt` must use `find_package(OpenCV REQUIRED)`)
> - File conversion is done via `std::system()` calls to `magick` / `gs`
> - Color results must include a `toHex()` method converting BGR → RGB hex string
> - Throw `std::runtime_error` with a descriptive message if conversion fails or the file format is unsupported
> - The tmp PNG files should be written to a `./tmp/` directory, created automatically if missing (`std::filesystem::create_directories`)
>
> **Deliverables:**
> - `converter.h` — file-to-PNG conversion logic
> - `color_extractor.h` — K-Means color extraction and background filtering
> - `main.cpp` — CLI entry point with formatted table output
> - `CMakeLists.txt`
> - `README.md` with build instructions and example usage
